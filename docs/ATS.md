# ATS

## Summary

Applicant Tracking Systems are natorious for throwing out perfectly good applications. Below is some guidlines that are used when making this project.

## Output

PDFs are the gold standard for ATS systems. It's important to remember that it should be an actual PDF and not an image of a PDF.

## Fonts

Printing to *.pdf only should be in one of these four fonts as they are the easiest to read for an ATS.

- Arial
- Calibri
- Times New Roman
- Helvetica

## Formatting

The format is meant to be intentionally boring. No human will ever look at it. It's meant to simply get past the bots to approve and pass on to the human to call you. Focus on making yourself exciting.

Additional formatting like text boxes, absolute positions, multiple columns of data, etc. can cause problems for the ATS and it will throw out your application.

Custom headers are probably never going to be supported because again, this is meant to be pretty straight forward in terms of output.

## Image/Icon/Unicharacter support

This does not support a tool like a rich text editor specifically for the reason that adding additional formatting or adding images, icons, or characters which aren't common anongst the spoken language to be useful and can cause issues.
